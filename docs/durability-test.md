# Phase 6 実機耐久試験

## 目的

常設ホストが 24 時間以上安定稼働し、Wi-Fi 切断後に自動復旧することを確認する。

## 事前準備

- 常設 iPhone / iPad / Mac を給電し続ける
- ホストアプリを起動したままにする
- 同一 Wi-Fi に参加者端末を 1 台以上接続

## ホストアプリからの確認

常設画面メニュー → **耐久・メトリクス** で以下が可能です。

- サーバー稼働時間・配信回数の確認
- **セルフテストを実行**（`POST /api/durability/self-test`）
- 耐久ログの閲覧・クリア（UserDefaults `jukebox.durability.events`）

## 手順

1. ホスト起動直後にセルフテストを実行し `passed: true` を確認
2. 24 時間放置（再生・キュー追加を時々実施）
3. ルーターを 30 秒オフ → オンにして Wi-Fi 復旧を確認
4. アプリをバックグラウンド → フォアグラウンド復帰でサーバーが継続することを確認

## 合格基準

| 項目 | 基準 |
|------|------|
| セルフテスト API | `passed: true`、状態取得 1000ms 未満 |
| 24h クラッシュ | 0 回 |
| セッション復旧 | 再起動後もキュー/再生位置が復元 |
| Wi-Fi 復旧 | 60 秒以内に `network_restored` ログ |
| ウォッチドッグ | 30 秒ごとに `watchdog_tick` が記録 |

## 記録テンプレート

```
試験日:
端末:
OS:
結果:
network_restored 回数:
watchdog_tick 件数:
備考:
```
